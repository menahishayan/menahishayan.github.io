# Monorepos in the Real World  
**GeekyAnts — May 2023, Bengaluru, India**

At the GeekyAnts developer event, I expanded on the technical and organizational aspects of building with monorepos, especially in mixed-platform environments where React and React Native coexist. The session focused on lessons learned while scaling codebases across web and mobile teams.

## Talk Overview

Rather than advocating blindly for monorepos, the talk focused on how to design one well—and when it’s actually the right choice.

**Topics Covered**
- Recap of traditional repo structures and why they start to break down
- Core tooling: Yarn Workspaces, Lerna (where it helps, where it doesn't)
- Handling cross-platform modules, shared business logic, and design tokens
- Deep dives:
  - Dealing with tightly coupled modules across packages
  - Shared TypeScript configs and linting setups
  - Managing scoped dependencies and avoiding cascade failures
  - Bundler quirks when packaging for different platforms
- Techniques to keep monorepos maintainable with growing teams
- Cultural side of monorepos: how ownership and reviews should evolve

## Key Points
- Monorepos are powerful, but bring coordination overhead
- Tooling alone isn’t enough—structure, convention, and documentation matter more
- Cross-platform packages require careful abstraction and standardization
- Effective monorepos scale both code and teams if executed thoughtfully
