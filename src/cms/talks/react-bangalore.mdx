# Production-Grade Monorepos  
**React Bangalore — June 2023, Bengaluru, India**

At React Bangalore 2023, I spoke about the practicalities of setting up and managing monorepos for frontend teams working with React and React Native. The talk was shaped by direct experience leading engineering teams through monorepo migrations and scale-ups.

## Talk Overview

The session broke down not just the *why* behind monorepos, but also the *how*—covering technical nuances, tooling decisions, and team workflows.

**Topics Covered**
- Trade-offs between traditional multi-repo setups and monorepos
- Using Yarn Workspaces for dependency management
- Structuring shared code: utils, types, and cross-platform modules
- Common pain points:
  - Deeply entangled shared files
  - Linter and formatter config reuse
  - TS/JS interoperability
  - Bundler quirks and optimization
- Case studies from production: where monorepos improved DX, and where they didn’t
- Criteria for when *not* to use a monorepo
- Final thoughts on maintaining healthy repo hygiene at scale

## Key Points
- Monorepos improve visibility, tooling, and coordination—when done right
- Thoughtful structure and clear ownership boundaries are essential
- Developer productivity gains often depend on how well edge cases are handled
- Not every team or project benefits from a monorepo—context matters
